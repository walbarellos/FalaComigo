## Ciclo: Frontend P0 — Prancha Acessível para Neurodivergentes, Pais, Mães e Cuidadores

**Objetivo do ciclo:** criar a primeira estrutura frontend realmente executável do app, com foco absoluto em comunicação simples, previsível, offline, acessível e impossível de confundir no uso cotidiano.

**Stack soberana assumida:** Android nativo + Kotlin + Jetpack Compose.

**Raiz técnica sugerida:**


**Endereço:**

```text

```

**Funcionalidade:**
Ponto de entrada do app. Deve iniciar o tema global, aplicar configurações de acessibilidade, carregar o `AppNavHost` e abrir diretamente no modo comunicação, sem login, sem cadastro e sem onboarding obrigatório bloqueante.

**Prompt ideal para a Vida Relacional:**

# Gate de aceite deste ciclo

Este ciclo só deve ser considerado fechado quando o app conseguir cumprir isto:

```text
Ao abrir o app, a pessoa vê uma prancha clara.
Ao tocar em um símbolo, o app tenta falar.
Se a voz falhar, o app não quebra.
A prancha urgente existe.
O editor não aparece por acidente.
Os botões são grandes.
A interface é calma.
A mãe, o pai ou cuidador entendem sem manual.
A pessoa neurodivergente não precisa entender o sistema para se comunicar.
```

**Frase soberana do ciclo:**

```text
O frontend do Fala Comigo não deve parecer um software.
Deve parecer uma ponte calma, confiável e imediata entre a necessidade da pessoa e o mundo que precisa ouvi-la.
```

Login no app é algo incômodo que deveria ser opcional e não obrigatório.

Perfeito. Isso deve virar **regra soberana de produto**.

## Decisão atualizada

**Login no Fala Comigo deve ser opcional, nunca obrigatório.**

A pessoa neurodivergente, a mãe, o pai ou cuidador deve conseguir:

```text
abrir o app
ver a prancha
tocar no símbolo
falar
usar urgente
editar localmente
usar offline
```

sem criar conta, sem e-mail, sem senha, sem internet e sem aceitar fluxo complexo.

## Regra para o projeto

```text
Nenhuma função essencial de comunicação pode depender de login.
```

## O que pode ficar sem login

```text
Pranchas principais
Prancha urgente
TTS
Modo frase
Editor local
Símbolos locais
Configurações de acessibilidade
Configurações de voz
Dados salvos no aparelho
Uso offline
```

## O que pode exigir login futuramente, se existir

```text
Backup em nuvem
Sincronização entre celulares
Compartilhamento familiar
Restauração de dados
Acesso de profissional autorizado
Histórico multiusuário
```

Mesmo assim, deve ser apresentado como:

```text
Criar conta é opcional.
O app funciona normalmente sem conta.
```

## Correção na lista sequenciada

Onde aparecer `login`, `auth`, `conta`, `cadastro`, `session`, `usuário autenticado`, deve ser tratado como **fora do MVP**.

O termo correto para o ciclo atual é:

```text
Acesso local sem conta
```

E não:

```text
Login
Autenticação obrigatória
Cadastro inicial
Conta de usuário
```

## Frase soberana

```text
O Fala Comigo não pode colocar uma senha entre a pessoa e a própria voz.
```
